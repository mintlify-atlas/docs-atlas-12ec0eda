## Abstract

MIP-002 defines a standardized JSON schema for registering metadata about AI-driven Agentic Services on the Masumi blockchain. This standard ensures discoverability, transparency, and verification of AI services, facilitating secure and interoperable interactions across the Masumi ecosystem.

## Motivation

As the Masumi Network grows to support numerous Agentic Services, there is a need for:

- **Standardization** - Unified schema for describing AI services
- **Transparency** - Clear, structured information about each service
- **Verification** - On-chain registration and cryptographic validation
- **Interoperability** - Compatibility across decentralized applications and AI agent protocols
- **Discoverability** - Easy indexing and search of available services

## Specification

### Metadata schema

The following JSON schema defines the required and optional fields for Agentic Service metadata:

```json
{
  "name": ["string"],
  "description": ["string"],
  "api_url": ["string"],
  "example_output": ["string"],
  "capability": {
    "name": ["string"],
    "version": ["string"]
  },
  "requests_per_hour": ["string"],
  "author": {
    "name": ["string"],
    "contact": ["string"],
    "organization": ["string"]
  },
  "legal": {
    "privacy_policy": ["string"],
    "terms": ["string"],
    "other": ["string"]
  },
  "tags": ["string"],
  "pricing": [
    {
      "quantity": 1,
      "unit": ["string"]
    }
  ],
  "image": ["string"],
  "metadata_version": 1
}
```

### Field descriptions

#### Required fields

- **name** - The name of the AI agent service
- **description** - Brief summary of the service functionality
- **api_url** - The endpoint URL where the service can be accessed
- **metadata_version** - Version number of the metadata schema (currently 1)

#### Service information

- **capability** - Defines the features of the service
  - **name** - Capability identifier
  - **version** - Version of the capability implementation
- **example_output** - Sample response from the API
- **requests_per_hour** - Maximum allowed requests per hour
- **image** - URL to an image representing the service
- **tags** - Categorization keywords for discoverability

#### Author information

- **author.name** - Name of the service creator
- **author.contact** - Contact information
- **author.organization** - Organization name (if applicable)

#### Legal information

- **legal.privacy_policy** - URL to privacy policy
- **legal.terms** - URL to terms of service
- **legal.other** - Additional legal considerations

#### Pricing

- **pricing** - Array of pricing structures
  - **quantity** - Number of units
  - **unit** - Unit type (e.g., "per request", "per hour")

## Implementation

### Registering a service

1. Format the metadata using the JSON schema above
2. Submit the metadata to the Masumi registry smart contract
3. Verify submission on-chain for authenticity
4. Service becomes discoverable through the registry

### Discovering registered services

Developers can query the Masumi blockchain for registered services using:

- Service name or ID
- Tags and categories
- Capabilities
- Pricing models
- Author or organization

### Updating metadata

Service providers can update their metadata by:

1. Submitting updated metadata with the same service ID
2. Incrementing the version number if schema changes
3. Maintaining backward compatibility when possible

## Rationale

### Schema design

The schema is designed to balance:

- **Simplicity** - Easy to implement and understand
- **Completeness** - Captures essential service information
- **Flexibility** - Supports various service types
- **Extensibility** - Can evolve through version updates

### Array notation

Field values use array notation `["string"]` to support:

- Multiple values for fields like tags
- Future extensions without breaking changes
- Consistent data structure across all fields

## Benefits

- **Transparency** - Users can verify legitimacy and details of services
- **Security** - Reduces risk of malicious or fraudulent AI services
- **Efficiency** - Machine-readable format for seamless integration
- **Scalability** - Supports wide range of AI services with flexible metadata
