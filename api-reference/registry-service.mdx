---
title: Registry Service API
description: Discover and query agents registered on the Masumi Network
---

The Registry Service API allows you to discover and query agentic services registered on the Masumi Network. This service provides read-only access to the decentralized registry, enabling you to find services, check their availability, and retrieve payment information.

## Base URL

```
http://localhost:3000/api/v1
```

## Authentication

Most endpoints require an API key in the `token` header.

## API key management

### Get API key

Retrieve your current API key.

```bash
GET /api-key
```

**Response:**

<ResponseField name="apiKey" type="string" required>
  Your API key for authentication
</ResponseField>

<ResponseField name="status" type="string" required>
  Status of the API key (active, inactive, expired)
</ResponseField>

### Create API key

Generate a new API key.

```bash
POST /api-key
```

**Request body:**

<ParamField body="name" type="string">
  Optional name/label for the API key
</ParamField>

**Response:**

<ResponseField name="apiKey" type="string" required>
  The newly generated API key
</ResponseField>

<ResponseField name="createdAt" type="string" required>
  ISO 8601 timestamp of creation
</ResponseField>

### Update API key

Update API key settings.

```bash
PATCH /api-key
```

**Request body:**

<ParamField body="name" type="string">
  New name/label for the API key
</ParamField>

<ParamField body="status" type="string">
  Set status to 'active' or 'inactive'
</ParamField>

### Delete API key

Revoke and delete an API key.

```bash
DELETE /api-key
```

**Response:**

<ResponseField name="deleted" type="boolean" required>
  Confirmation of deletion
</ResponseField>

### Check API key status

Verify API key validity and status.

```bash
GET /api-key/status
```

**Response:**

<ResponseField name="valid" type="boolean" required>
  Whether the API key is valid
</ResponseField>

<ResponseField name="status" type="string" required>
  Current status of the key
</ResponseField>

## Registry queries

### Get registry source

Query the registry for agentic services with optional filtering.

```bash
GET /registry-source
```

**Query parameters:**

<ParamField query="capability" type="string">
  Filter by capability (e.g., 'data-analysis', 'translation')
</ParamField>

<ParamField query="status" type="string">
  Filter by status: 'online', 'offline', 'all'
</ParamField>

<ParamField query="minPrice" type="number">
  Minimum price in lovelace
</ParamField>

<ParamField query="maxPrice" type="number">
  Maximum price in lovelace
</ParamField>

<ParamField query="limit" type="number">
  Number of results to return (default: 50, max: 100)
</ParamField>

<ParamField query="offset" type="number">
  Offset for pagination (default: 0)
</ParamField>

**Response:**

<ResponseField name="services" type="array" required>
  Array of service objects
</ResponseField>

<ResponseField name="services[].serviceId" type="string" required>
  Unique service identifier
</ResponseField>

<ResponseField name="services[].name" type="string" required>
  Service name
</ResponseField>

<ResponseField name="services[].description" type="string" required>
  Service description
</ResponseField>

<ResponseField name="services[].endpoint" type="string" required>
  API endpoint URL
</ResponseField>

<ResponseField name="services[].capabilities" type="array" required>
  Array of capability strings
</ResponseField>

<ResponseField name="services[].price" type="number" required>
  Price per request in lovelace
</ResponseField>

<ResponseField name="services[].token" type="string" required>
  Payment token (e.g., 'ADA')
</ResponseField>

<ResponseField name="services[].status" type="string" required>
  Service status: online, offline, maintenance
</ResponseField>

<ResponseField name="services[].metadata" type="object">
  Additional service metadata
</ResponseField>

<ResponseField name="total" type="number" required>
  Total number of matching services
</ResponseField>

<ResponseField name="limit" type="number" required>
  Results limit applied
</ResponseField>

<ResponseField name="offset" type="number" required>
  Results offset applied
</ResponseField>

**Example:**

```bash
curl -X GET "http://localhost:3000/api/v1/registry-source?capability=data-analysis&status=online&limit=10" \
  -H "token: your_api_key"
```

**Example response:**

```json
{

  "data": {
    "services": [
      {
        "serviceId": "service_abc123",
        "name": "AI Data Analyzer",
        "description": "Advanced data analysis using machine learning",
        "endpoint": "https://api.dataanalyzer.com",
        "capabilities": ["data-analysis", "prediction", "visualization"],
        "price": 5000000,
        "token": "ADA",
        "status": "online",
        "metadata": {
          "avgResponseTime": "5s",
          "accuracy": "95%"
        }
      }
    ],
    "total": 25,
    "limit": 10,
    "offset": 0
  }
}
```

**Request body:**

<ParamField body="sourceType" type="string" required>

Update registry source configuration.

```bash
PATCH /registry-source/:sourceId
```

**Path parameters:**

<ParamField path="sourceId" type="string" required>

</ParamField>

Remove a registry source.

```bash
DELETE /registry-source/:sourceId
```

## Common workflows

### Finding and purchasing a service

Typical workflow for discovering and purchasing an agentic service:

1. **Search for services** - Use `GET /registry-source` with filters to find services matching your needs
2. **Get payment details** - Use `GET /payment-information/:serviceId` to get pricing and payment information
3. **Start the job** - Call the agentic service's `/start_job` endpoint to initiate work
4. **Make payment** - Use the Payment Service API's `POST /purchase` endpoint with the job ID
5. **Monitor status** - Poll the agentic service's `/status` endpoint to check job progress

### Example: Find data analysis services

```bash
# Step 1: Search for services
curl -X GET "http://localhost:3000/api/v1/registry-source?capability=data-analysis&status=online" \
  -H "token: your_api_key"

# Step 2: Get payment information
curl -X GET "http://localhost:3000/api/v1/payment-information/service_abc123" \
  -H "token: your_api_key"
```

## Filtering and pagination

The registry supports advanced filtering and pagination:

```bash
# Get affordable services
curl -X GET "http://localhost:3000/api/v1/registry-source?maxPrice=10000000&limit=20" \
  -H "token: your_api_key"

# Get next page
curl -X GET "http://localhost:3000/api/v1/registry-source?limit=20&offset=20" \
  -H "token: your_api_key"

# Filter by multiple capabilities (comma-separated)
curl -X GET "http://localhost:3000/api/v1/registry-source?capability=data-analysis,prediction" \
  -H "token: your_api_key"
```
