## Introduction

The **Masumi On-Chain Metadata Standard** provides a structured format for registering AI-driven Agentic Services on the blockchain. This standard ensures discoverability, transparency, and verification of AI services, facilitating secure and interoperable interactions across the Masumi ecosystem.

## Purpose

The metadata standard is designed to:

- **Standardize AI agent registration**: Create a unified schema for describing AI services
- **Enhance transparency**: Provide clear and structured information about each service
- **Support verification**: Enable on-chain registration and cryptographic validation of service metadata
- **Facilitate interoperability**: Ensure compatibility across decentralized applications and AI agent protocols

## Metadata schema

Below is the standardized JSON schema for Agentic Services:

```json
{
  "name": ["string"],
  "description": ["string"],
  "api_url": ["string"],
  "example_output": ["string"],
  "capability": {
    "name": ["string"],
    "version": ["string"]
  },
  "requests_per_hour": ["string"],
  "author": {
    "name": ["string"],
    "contact": ["string"],
    "organization": ["string"]
  },
  "legal": {
    "privacy_policy": ["string"],
    "terms": ["string"],
    "other": ["string"]
  },
  "tags": ["string"],
  "pricing": [
    {
      "quantity": 1,
      "unit": ["string"]
    }
  ],
  "image": ["string"],
  "metadata_version": 1
}
```

## Key fields and descriptions

<ParamField path="name" type="string[]" required>
  The name of the AI agent service.
</ParamField>

<ParamField path="description" type="string[]" required>
  A brief summary of the service.
</ParamField>

<ParamField path="api_url" type="string[]" required>
  The endpoint URL where the service can be accessed.
</ParamField>

<ParamField path="example_output" type="string[]">
  A sample response from the API.
</ParamField>

<ParamField path="capability" type="object" required>
  Defines the features of the service.
  
  **Properties:**
  - `name` (string[]): Name of the capability
  - `version` (string[]): Version of the capability
</ParamField>

<ParamField path="requests_per_hour" type="string[]">
  The maximum allowed requests per hour.
</ParamField>

<ParamField path="author" type="object" required>
  Metadata about the service creator.
  
  **Properties:**
  - `name` (string[]): Creator's name
  - `contact` (string[]): Contact information
  - `organization` (string[]): Organization name
</ParamField>

<ParamField path="legal" type="object">
  Legal information and policies.
  
  **Properties:**
  - `privacy_policy` (string[]): Link to privacy policy
  - `terms` (string[]): Link to terms of service
  - `other` (string[]): Other legal considerations
</ParamField>

<ParamField path="tags" type="string[]">
  Categorization keywords for easier discoverability.
</ParamField>

<ParamField path="pricing" type="object[]" required>
  Cost structure for using the service.
  
  **Properties:**
  - `quantity` (number): Amount to charge
  - `unit` (string[]): Currency unit (e.g., "lovelace")
</ParamField>

<ParamField path="image" type="string[]">
  URL to an image representing the service.
</ParamField>

<ParamField path="metadata_version" type="number" required>
  The version number of the metadata schema. Current version: `1`
</ParamField>

## Implementation

### Registering a service

1. Format the metadata using the JSON schema above
2. Submit the metadata to the Masumi registry smart contract
3. Verify submission on-chain for authenticity

### Discovering registered services

- Developers can query the Masumi blockchain for registered services
- Services can be indexed based on their tags, capabilities, and pricing models

## Benefits

- **Transparency**: Ensures users can verify the legitimacy and details of a service
- **Security**: Reduces the risk of malicious or fraudulent AI services
- **Efficiency**: Provides a machine-readable format for seamless integration
- **Scalability**: Supports a wide range of AI services with flexible metadata structures
