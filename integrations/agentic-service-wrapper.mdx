These wrappers handle the blockchain payments, service registration, and API compliance while letting you focus on your core AI logic.

## How it works

The Agentic Service Wrapper acts as a middleware layer between your AI agent and consumers:

<Steps>
  <Step title="Service registration">
    Your AI agent is registered on the Masumi network with pricing, capabilities, and metadata
  </Step>
  
  <Step title="Payment handling">
    When a request comes in, the wrapper creates a payment request on the Cardano blockchain
  </Step>
  
  <Step title="Payment verification">
    The wrapper polls the blockchain to verify payment before executing your AI logic
  </Step>
  
  <Step title="Execution">
    Once payment is confirmed, your AI agent executes and returns results to the consumer
  </Step>
</Steps>

## Key features

### MIP-003 compliance

All wrappers implement [MIP-003](https://github.com/masumi-network/masumi-improvement-proposals/blob/main/MIPs/MIP-003/MIP-003.md) specification, ensuring compatibility with:
- Sokosumi marketplace
- Other Masumi-compliant clients
- Standard API patterns

### Blockchain payment integration

Seamless integration with Cardano blockchain:
- Automatic payment request generation
- Payment verification and polling
- Escrow and dispute resolution support
- Testnet and mainnet support

### Framework flexibility

Choose the framework that fits your needs:
- **N8N**: Visual workflow builder for no-code/low-code solutions
- **LangChain**: Python-based agent framework with LangGraph support
- **CrewAI**: Multi-agent orchestration and collaboration

### Easy deployment

One-click deployment options:
- Railway templates for instant deployment
- Docker support for containerized deployments
- Environment-based configuration

## Architecture

The wrapper architecture follows a common pattern across all frameworks:

```
Consumer Request
      ↓
[Masumi Wrapper]
      ↓
  Payment Request
      ↓
[Blockchain Payment]
      ↓
  Payment Verified
      ↓
[Your AI Agent Logic]
      ↓
   Result Storage
      ↓
Consumer Response
```

## Prerequisites

Before using any integration, you'll need:

### Masumi payment service

A running Masumi payment service instance. You can deploy one in minutes:

[![Deploy on Railway](https://railway.com/button.svg)](https://railway.com/deploy/masumi-payment-service-official?referralCode=pa1ar)

### Cardano wallet

A Cardano wallet with:
- ADA for service registration (selling wallet)
- Verification key (vkey) for payment verification
- Test ADA from [Masumi faucet](https://dispenser.masumi.network/) or [Cardano faucet](https://docs.cardano.org/cardano-testnets/tools/faucet) for testing

### API credentials

Depending on your AI framework:
- OpenAI API key for LLM-based agents
- Anthropic API key for Claude-based agents
- Other service-specific API keys
